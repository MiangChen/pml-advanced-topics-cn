# 讲解风格规范（Style Guide）——「乐高积木」风格

> 把这份文件连同某一节的原文一起喂给任何 AI，它就能产出风格一致的中文讲解。
> 适用：Kevin Murphy《Probabilistic Machine Learning: Advanced Topics》(MIT Press, 2023) 第四部分「生成模型」(第 20–26 章)。
> 标准样板：`风格实验/20.1-20.2_生成模型的类型_乐高版.md`。

---

## 0. 一句话定位

**一本难书 = 一盒看着吓人的乐高。我们不一上来就给读者看盒子上那张复杂成品图，而是先把零件倒出来，从最小的一块讲起，一块一块往上扣，每扣一块都让读者看得见它扣在哪、为什么扣，最后读者抬头一看——那个复杂大家伙，是他自己拼出来的。**

读者设定：**聪明但零基础**——可以是个小朋友。不许预设他懂任何术语。

---

## 1. 七条拼装规则（核心）

1. **从最小的积木块开始**：每节/每章的第一句不许有任何术语，先建一个连小朋友都懂的最底层直觉，作为"底板"。
2. **一次只发一块积木**：每个小标题只引入**一个**新概念，且只能用**前面已拼好的**零件来解释——绝不依赖还没讲的东西。
3. **每块都看得见"扣在哪"**：引入新概念时，明说"它扣在刚才那块的哪个位置、补上了上一块留下的什么缺口"。
4. **从具体到抽象，符号最后才出现**：先大白话和画面感，直觉立住后再说"数学家把这个记作 …"。符号是给已懂的东西**贴标签**，不是吓人。
5. **复杂 = 简单积木的堆叠**：再唬人的模型，都拆成"它不过是 A 块 + B 块 + C 块拼起来的"。
6. **每隔几块，退一步看半成品**（🔭）：阶段性"抬头看一眼"已拼出的形状；每章/节末用"成品揭幕"回收全部积木。
7. **100% 不丢件**：原书每个知识点都必须是成品里的一块积木——可换朴素说法、可重排拼装顺序，但**一块都不能少**（含脚注、交叉引用）。

---

## 2. ⚠️ 硬规则：公式 1:1 全搬，绝不省略

通俗化**不等于**砍公式。原书出现的**每一个公式、符号、记号都必须 1:1 出现在讲解里**，否则读者只记住了比喻、没记住数学，等于白讲。

- 行内符号用 `$...$`，独立公式用 `$$...$$`，**完整保留原书写法与符号**（$p(x), p(x\mid c), z_1,\dots,z_L, q_\phi(z\mid x), L\ll D$ 等）。
- 做法：先用大白话/画面感讲直觉（规则 4），**然后紧跟一句"数学家记作 $\dots$"把公式贴上去**，并逐符号解释"这个字母代表刚才说的哪块积木"。
- 图、表里出现的函数与符号（如 $E(x), D(x), G(z), f^{-1}(z), x^d$）也算公式，同样要搬进正文解读。
- 自检：写完对照原文，确认**没有一个公式被"翻译成纯文字"而消失**。

---

## 3. 文件与切分规则

- **颗粒度**：原书每个 `X.Y` 小节 = 一个 `.md` 文件；极短节（一两段）可与下一节合并。
- **命名**：`<节号>_<中文标题>.md`（合并时 `20.1-20.2_xxx.md`）。
- **配图**：放 `figs/`，命名 `fig<编号>_<中文短名>.png`，正文相对路径引用。提取：`pdftotext -f <起> -l <止> book2.pdf -`；裁图：`pdftoppm -png -r 150` + PIL crop。

---

## 4. 文件结构模板

```
# 第 X 章 §X.Y —— 像拼乐高一样搞懂「<节标题>」
> 元信息块：哪本书、第几节、原书页码、声明"100%不丢知识点、公式1:1搬运"。
---
## 🧱 第 0 块（底板）：<最底层、零术语的直觉>
## 🧱 第 1 块：<一个新概念> + 紧跟其后的公式 $...$
## 🧱 第 2 块：……（每块一个概念，扣在前面）
> 🔭 退一步看半成品：……
…
## 🏁 成品揭幕：用 N 句话回收全部积木
> ➡️ 下一节 §X.Y+1 预告一句
```

---

## 5. 语言与标记

- **画面感优先**：抽象词一律换成画面（潜变量=一把"种子"；扩散=把图泼成雪花再擦回；GAN=造假者 vs 鉴宝师；能量=别扭分）。
- **emoji 克制标记**：`🧱`每块积木标题 / `🔭`退一步看半成品 / `💡`重要直觉 / `⚠️`坑与局限 / `📌`脚注或细节澄清 / `➡️`下节预告 / `🏁`成品揭幕。
- **对比成表**：任何"多者对比"用 Markdown 表格；原书表格翻译保留并补"为什么在乎/感觉"列。
- **保留交叉引用**：原文"见第21章/图25.10"照样中文化保留。

---

## 6. 反例（不要这样）

- ❌ 一上来就抛术语和定义。
- ❌ 为了通俗**砍掉或只用文字代替公式**（违反硬规则 §2）。
- ❌ 引入还没讲过的零件来解释新概念。
- ❌ 把对比写成大段文字而不用表格。
- ❌ 比喻/emoji 泛滥喧宾夺主，或为简洁删掉原文知识点、脚注、交叉引用。
